QuestionGame - Client-Server game

Introduction

Last year (2023) I stumbled across a game called "The Test". It is a game where 2 players take rounds and answer questions. For example player 1 answers first 2 queestions with the answers that appeals to them the most. After that second player first answers what they think player 1 answered in 2 previous questions and if he is correct they get 1p. for each correct answer. After that player 2 answers 2 new questions. In the next round player 1 first sees what player 2 thought he would answer and after that game goes as stated before. At that time I thought it would be a nice project to try to do this and that it would be a nice way to practice networking in Java.

Documentation

When I was creating this project it didn't grow as I expected, because it didn't grow it terms of number of files, but it grew it terms of number of commands I had to remember and what were they doing, so I thought that it would be nice to document this project, so that if I ever come back to it I will know what is going on (at least when it comes to logic of the program). I spent a lot of time trying to document it, making it complient with JavaDoc and adding multiple README.md files and as I look at it now, it was a very good idea as every function has a description what is it and its parameters are doing and even after few months it is much easier to go back to what I had in my mind at that time

Config files

When this project was evolving I wanted to test multiple of its features and with time it was obvious for mee that I had to change too many things in my 'main' to test it, so i came up with the idea of doing my configuration file.Both 'Server' and the 'Client' first initialized config handler to adjust some of the parameters so it would be easy for me to set something up or test it.

Client Side

Client side had to do only few things. 1.Create multiple clients 2. Sent messages to the Server from the chosen client. As I added more functionality to the server, I also needed some way of automating testing new commands, so I added it to the Client. Event though it is very basic it speed up my process of testing commands a lot. Client had to take care only of the few types of messages from the Server.

Server Side

Server side of this project was much more complicated than the Client side as it had to handle multiple clients and possibly multiple messages at the same time, where each message could have asked the server for a different thing to do. It is a basic Java Socket Server. It handles message processing such as getting user or the question from the database or adding them to it.

Data Storage

I learned using SQL and NoSQL in the past, so I decided not to use them but use simple file storage. I just wrote everything from and to a file and called it a database. I knew from the beginning that conventional SQL and even NoSQL databases (which are much slower than SQL) will be faster than this but it, but I also thought that it would be fun to implement everything by myself
