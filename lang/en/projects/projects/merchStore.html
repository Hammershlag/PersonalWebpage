MerchStore: A Comprehensive E-commerce Platform

Introduction

MerchStore is a comprehensive e-commerce platform developed using Java and Spring Boot for the backend, and JavaScript for the frontend. It is designed to provide a seamless online shopping experience, offering a wide range of features that cater to both buyers and sellers. For buyers, the application offers a user-friendly interface to browse items, add them to their cart, and proceed to checkout. It also provides a personalized shopping experience by remembering the user's browsing history and showing them their most recently viewed items. The application also supports a cart system where users can add items they wish to purchase, and when ready, they can proceed to a secure checkout process. For sellers, the application provides an easy-to-use platform to list their items for sale. They can add detailed descriptions, set prices, and offer discounts. The application also features a robust advertisement system with different statuses (active, inactive, deleted), allowing sellers to manage their listings effectively. The application also integrates with an external API to fetch and update the current exchange rates, ensuring that all transactions are carried out with the most recent and accurate exchange rates. This feature is particularly useful for international transactions, providing users with the most accurate pricing in their local currency.

Authentication

Security is a top priority in MerchStore. The application provides secure login and registration functionalities, using advanced encryption techniques to protect user data. It also supports different roles for users, including owner, admin, and user, each with their own set of permissions and access levels. This role-based access control ensures that sensitive operations are only accessible to authorized users.

Exchange Rate API

The application integrates with an external API to fetch and update the current exchange rates. This feature ensures that all transactions are carried out with the most recent and accurate exchange rates. It provides users with real-time pricing information, making it easier for them to make informed purchasing decisions.

Error Handling

The application includes a custom error page to handle any unexpected errors or exceptions. This ensures a smooth user experience even when something goes wrong. The error handling mechanism is designed to provide informative error messages, helping users understand what went wrong and how they can resolve the issue.

E-commerce Features

MerchStore offers a wide range of e-commerce features designed to enhance the online shopping experience. Users can browse items, add them to their cart, and proceed to checkout to make a purchase. The application also keeps track of the user's browsing history, showing them their most recently viewed items. Sellers can list their items for sale, add detailed descriptions, set prices, and offer discounts. The application also supports a robust advertisement system, allowing sellers to manage their listings effectively.

Cookies

The application uses cookies to enhance the user experience. Cookies are used to remember the user's login information, personalizing the user interface based on their preferences, and tracking their browsing history. This allows MerchStore to provide a more personalized and convenient shopping experience.

Key Features

  • User Authentication: Login and registration functionalities.
  • User Roles: Distinct roles for owner, admin, and users, controlling access and permissions.
  • Product Management: Users can browse items, add them to their cart, and make purchases.
  • Exchange Rate API: Fetches and updates current exchange rates.
  • Error Handling: Includes a custom error page for handling unexpected errors or exceptions.
  • Cookies: Enhances user experience by remembering user information and personalizing the user interface.

Challenges and Learning

Developing MerchStore was a significant learning experience. It involved dealing with various challenges such as implementing secure user authentication, integrating with an external API, handling errors effectively, and managing user roles. Despite these challenges, the project was a great opportunity to learn and apply various technologies and practices such as Java, Spring Boot, JavaScript, Docker, PostgreSQL, and more.
